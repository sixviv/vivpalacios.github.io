Case Study Couchsurfing

Host availability is down
20% in Paris. Why?

Couchsurfing saw a 20% drop in host availability in Paris — fewer new sign-ups and reduced engagement from existing hosts. The task: diagnose the root cause, define the critical issue, and design a product response.

Approach

Started with a PESTEL analysis, following threads that had clear causal evidence rather than surface-level correlation. Three root cause buckets emerged:

  • Members less aligned with the platform's foundational values — creating friction and bad host experiences
  • Increased cost of living driving some hosts toward commercial alternatives (Airbnb)
  • Regulatory pressures in Paris limiting short-term hosting

Critical Issue

Of the three root causes, the one product can directly address is the values misalignment.

Hosts are burning out from mismatched, low-effort guests who don't share the spirit of cultural exchange — they just want a free bed. Hosts are the backbone of Couchsurfing: if they don't feel safe, respected, or culturally aligned, they leave. Low-quality guest requests create emotional friction that erodes trust in the platform over time.

Solution — Host Preference Controls

Give hosts tools to manage and filter guest requests based on their own preferences — reducing mismatched requests, lowering emotional burnout, and restoring a sense of safety and control.

Guest Request Preferences

Hosts filter incoming requests based on configurable criteria.

  • Minimum number of references
  • Languages spoken
  • Up to 5 shared interests
  • Countries visited
  • Show request match % to host

Pre-set Host Questions

Hosts define required questions guests must answer before submitting a request.

  • Up to 5 custom questions (CRUD)
  • Guest can't submit without answering
  • Host reviews answers before accepting
  • Screens out copy-paste requests
  • Confirms guest read host's profile

Launch Plan

Phase 1 — MVP (Limited Availability)

Guest request preferences, one open-text host question, request match % shown to hosts. Marketing re-engagement campaign: "You're in control again."

Phase 2 — GA

Up to 5 host questions, mutual auto-filter (optional), full rollout. Updated help center docs and support team training for smooth handoff.

Success Metrics

Primary

Higher request approval rate (requests → acceptances)

Primary

Decrease in host churn in Paris

Guardrail

No significant drop in total guest requests sent

Validated through soft launch A/B testing, surveys, and user interviews. Tracked via avg requests per host and acceptance ratio per high-volume city.

Long-term Idea

Couchsurfing Karma Program — recognize and celebrate hosts who embody the spirit of cultural exchange through badges, milestones, and non-monetary perks. Goal: create a sense of value, purpose, and social recognition for hosts without relying on financial incentives.
